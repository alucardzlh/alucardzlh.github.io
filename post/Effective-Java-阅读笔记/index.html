#Effective #java

Effective Java 阅读笔记

  1. build模式

  2. 通过构造器注入类，进行调用,而不是在内部直接创建类使用。

  3. 重用对象， 通过 static final 缓存昂贵实例 比如正则。

  4. 自动拆装箱 互转 会耗损更多性能 比如混用int 和integer。

  5. 轻量对象不要使用对象池

  6. 通过有效期、缓存大小、软弱引用来解决不能被GC释放的内存 weakHashMap

  7. 使用try with Resource

  8. Camparator 多参数数据排序 自反性 对称性 传递性 一致性 空

  9. 组合优于继承，接口优于抽象类 bird extends animal

    bird总是拥有fly animal又没有fly 导致基类庞大，子类多未使用方法

    问题：企鹅是鸟但不会飞，蝙蝠不是鸟但又需要飞，

    将fly能力作为一个组件对象 放到animal中 作为成员域 可选 并添加set方法

  10. 基于接口->抽象类->实现类 完成骨架实现。 公共逻辑或成员域写在抽象类层

  11. 用抽象类和抽象方法 类层次的方法，解决类中的if-else 标签类

  12. 基于11再递进，把具体行为抽象为接口，每个动物实例去实现它重写接口内执行方法。然后用animal去注入具体行为接口在类内硬重写方法调用行为接口内的方法。
